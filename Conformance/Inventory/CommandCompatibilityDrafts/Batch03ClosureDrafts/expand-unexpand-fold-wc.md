# Batch 03 Closure Draft: expand / unexpand / fold / wc

Scope: this draft covers only `expand`, `unexpand`, `fold`, and `wc`. It is a proposed update for Batch 03 review and does not edit `batch-03-text-streams.md`.

## `expand`

- Reference source checked: `References/LinuxSourceSnapshot/debian12-bookworm/sources/coreutils-9.1/src/expand.c`, especially `longopts`, `usage`, `expand`, and `main`; shared tab parsing from `expand-common.c`, especially `parse_tab_stops`, `validate_tab_stops`, `finalize_tab_stops`, `get_next_tab_column`, and `emit_tab_list_info`.
- Command-local implementation changed: `MSPExpandCommand.swift` and `MSPTextLayoutSupport.swift`.
- Implemented evidence: added GNU-style `--help` and `--version`; expanded obsolete numeric tab syntax to parse comma/blank separated tab lists with `/N` extend size and `+N` increment size; aligned the tab-stop model with `expand-common.c` for single `+N`/`/N`, explicit stops followed by final `/N` or `+N`, repeated specifier diagnostics such as `2,+4,8`, `/` plus `+` mutual exclusion, zero tab stops, too-large tab-stop numbers, and `specifier not at start of number`; retained existing `-i`, operands/stdin, CR/BS column handling, and byte-preserving local data path; added command-local stdin streaming that transforms chunks without preloading the whole stream. Modular tests cover default tabs, `-t`, obsolete `-N` and `-LIST`, `/N`, `+N`, `+4`, CR, invalid bytes, file operands, invalid character diagnostics, zero diagnostics, too-large diagnostics, `1/2`, repeated `+`, help, and chunked streaming state for `-i`.
- Safe oracle/capture candidates for coordinator: `expand --help | head -n 6`, `expand --version`, `printf 'a\tb\tc\n' | expand -t '2,/4'`, `printf 'a\tb\tc\n' | expand -t '2,+4'`, `expand -t '2,+4,8'`, `expand -t '1/2'`, `expand -t 0`, `printf 'a\tb\n' | expand -t '+4'`, `printf 'ab\rc\td\n' | expand -t 4 | od -An -t x1`, and invalid-byte input with one tab.
- Coordinator/shared actions: streaming file operands still need shared file-reader/output-writer support; exact locale display width and `SIZE_MAX`-distance diagnostics beyond Swift `Int` parsing remain shared portability work; external Debian oracle promotion is coordinator-owned.

Modular tests cover default initial blanks, `-a`, `-t`, obsolete `-N`, comma lists, `/+` grammar through shared parser, `+4`, CR, invalid bytes, `--first-only`, multi-file input, invalid character diagnostics, `/` plus `+` mutual exclusion, `specifier not at start of number`, version, and completed-line streaming writes.

- Coordinator/shared actions: streaming file operands still need shared file-reader/output-writer support; exact locale blank/display-width handling and `SIZE_MAX`-distance diagnostics beyond Swift `Int` parsing remain shared portability work; external Debian oracle promotion is coordinator-owned.

## `wc`

- Reference source checked: `References/LinuxSourceSnapshot/debian12-bookworm/sources/coreutils-9.1/src/wc.c`, especially `longopts`, `usage`, `write_counts`, `wc`, `get_input_fstatus`, `compute_number_width`, `main`, `--files0-from`, and `--debug`.
- Command-local implementation changed: `MSPWcCommand.swift`.
- Implemented evidence: added GNU-style `--help` and `--version`; added accepted `--debug` with Debian Core100 count behavior and no MSP-private stderr; added `--files0-from=F` for WorkspaceFS list files and `--files0-from=-` from stdin, including rejection of command-line operands with `--files0-from`, zero-length NUL records with line number, and the GNU special case that forbids `-` as a file name when the list is read from stdin; corrected multi-file total generation and count-column width for expanded `--files0-from` operands; fixed NUL/control bytes so they do not incorrectly start words, matching `wc.c` printable-character word logic. Modular tests cover stdin/dash/totals, byte-only formatting, wide/combining/tab display width, NUL bytes, very long line `-L`, chunk-boundary UTF-8 streaming, file totals, missing file, common count modes, `--files0-from`, mixed operand diagnostics, stdin-list `-` diagnostics, `--debug`, help, and version.
- Safe oracle/capture candidates for coordinator: `wc --help | head -n 8`, `wc --version`, `wc --debug -c`, `wc -l -w -c --files0-from=list0`, `printf -- '-\0' | wc --files0-from=-`, empty-record lists, list file with a missing member, invalid UTF-8 with `-m -c -L`, combining and wide characters, NUL bytes, very long lines, and multi-file spacing.
- Coordinator/shared actions: exact locale `iswspace`/word-boundary parity, invalid multibyte `-m` behavior, and display width policy need shared locale/text-width support; file operands and large `--files0-from` should eventually stream from WorkspaceFS instead of loading data/list content in memory; exact GNU `--debug` CPU/optimization messages are platform-specific diagnostics and can be sampled separately; external Debian oracle promotion is coordinator-owned.

# Batch 04 - Text Languages And Search

This is a pre-release compatibility audit matrix for Core100 command work. It is not a support statement. All "supported" notes below mean "accepted by the current Swift implementation", not "GNU/Linux compatible".

## awk

- Command: `awk`
- MSP implementation: `Implementations/Swift/Sources/MSPPOSIXCore/Commands/Text/Awk/MSPAwkCommand.swift` plus the custom parser/runner under `Implementations/Swift/Sources/MSPPOSIXCore/Commands/Text/Awk/`. The entry accepts `-F`, `-v`, `-f`/`--file`, `-W version`, `-W help`, and `-W usage`; it emits a `mawk 1.3.4 20200120` version string.
- Reference source: `References/LinuxSourceSnapshot/debian12-bookworm/sources/mawk-1.3.4-20200120/main.c` (`main -> initialize -> parse -> execute`), `init.c` (`initialize`, `process_cmdline`, `set_ARGV`, `load_environ`, `usage`), `scan.c` (`yylex`, `collect_RE`, string/regex escapes, `print_flag`, `getline_flag`), `parse.c` (`parse`, `yyparse` reductions for awk grammar and bytecode emission), and `execute.c` (`execute` VM opcodes such as `_NEXT`, `_NEXTFILE`, `_RANGE`, `_CALL`, `_EXIT`, `_RET`).
- GNU/Linux parameter surface: mawk 1.3.4 command line from `init.c` includes `-F value`, `-f file`, `-v name=value`, `--`, and `-W version`, `-W binmode`, `-W dump`, `-W help`, `-W interactive`, `-W exec file`, `-W random=number`, `-W sprintf=number`, `-W posix_space`, and `-W usage`. Language surface from `scan.c`/`parse.c`/`execute.c` includes `BEGIN`/record/`END`; pattern-only and action-only programs; expression, regex, boolean, and range patterns; variables such as `ARGC`, `ARGV`, `FILENAME`, `FNR`, `FS`, `NF`, `NR`, `OFMT`, `OFS`, `ORS`, `RS`, `RSTART`, `RLENGTH`, `SUBSEP`, `ENVIRON`, and `ERRNO`; scalar/array coercion; `if/else`, `while`, `do while`, C-style `for`, `for (k in a)`, `break`, `continue`, `next`, `nextfile`, `exit expr`, `return`, `delete element`, and `delete array`; user functions and local arguments; field references/assignments; regex/string/numeric/math/time builtins; `getline` from stdin/file/pipe; `print`/`printf` with `>`, `>>`, and pipe output; `close()`; and shell `system()`.
- Currently supported by MSP: Inline programs and `-f` program files; `-W version`, `-W help`, and `-W usage`; file operands and stdin; assignment operands after the program; `BEGIN`, record, and `END` blocks; user functions; `print`, `printf`, `delete`, `return`, `exit` without status, `if/else`, C-style `for`, `for in`, and `while`; arrays and field assignment; arithmetic/comparison/regex operators, ternary evaluation, increment/decrement, and compound assignment; `length`, `index`, `sprintf`, `substr`, `tolower`, `sub`, `gsub`, `gensub`, `match`, `split`, and `close`; file and pipe `getline`; output to `/dev/stdout`, `/dev/stderr`, `/dev/null`, and WorkspaceFS files. Streaming exists only when reading stdin directly or `-`.
- Closure status:
  - Implemented evidence: `MSPAwkCommand.swift` accepts `-F`, `-v`, `-f`/`--file`, `-W version`, `-W help`, and `-W usage`, loads script files through WorkspaceFS, routes command-output `getline` through `context.runCommandLine`, and streams stdin records for simple stdin/`-` cases. `MSPTextLanguageCommandOracleTests.swift` covers field sums, `-F`, help/usage, missing-file diagnostics, and invalid options; `MSPWorkerCRecordStreamCommandTests.swift` covers early `exit` closing upstream while still running `END`; Core100 oracle cases cover field sum, field separator, program file, and missing file. Vendored mawk source remains the broader reference surface.
  - Shared-surface deferred implementation: full mawk language equivalence is coordinator-level hard deferred, not because the options are cold, but because `scan.c`/`parse.c`/`execute.c` define a complete lexer/parser/bytecode VM with process and file side effects that cannot be made source-equivalent by a local command-file patch without shared policy, cancellation, and oracle promotion. Coordinator action covers the rest of the mawk `-W` family (`dump`, `exec`, `interactive`, `random`, `sprintf`, `posix_space`, `binmode`), exact option/`ARGV` construction, multi-file `FNR`/`FILENAME`, `ARGC`/`ARGV`/`ENVIRON`/`OFMT`/`CONVFMT`/`ERRNO`, `next`/`nextfile`/`break`/`continue`/`do while`/`exit status`, range patterns, full regex/string escape and `FS`/`RS` behavior, math/time/random builtins, `system()`, pipe output, exact pipe/file getline semantics, and scalable multi-file streaming.
  - Shared-surface oracle/stress plan: coordinator action must add safety-audited oracle cases for multi-file state, assignment ordering, complex regex and record separators, arrays/functions, getline variants, pipe output/system policy paths, exit status, huge records/arrays, long-running loops, and pipeline/redirection stress. Safe case draft: all file writes stay under case root; `system()`/pipe cases use harmless `printf`/`cat` only through MSP policy; long-loop cases must have timeout/cancellation assertions before VPS sampling.
  - Deferred/forbidden with reason: unmediated host process execution from `system()`, pipe getline, or pipe output is forbidden by MSP policy. Future implementation must route through `context.runCommandLine` plus active audit/cancellation and WorkspaceFS routing; direct `execvp`/host shell spawning is not allowed.
- Must implement: None after this batch; command-local implementation and tests are recorded in Closure status, and shared-runtime or policy-bound items are recorded under Deferred with reason.
- Deferred with reason: full mawk VM parity is hard deferred to coordinator/shared-language work because it crosses parser/runtime/policy boundaries; unimplemented language forms are recorded in the shared implementation plan.
- Forbidden by policy: Unmediated host command execution from `system()`, pipe getline, or pipe output is forbidden. If implemented, it must route through `context.runCommandLine` and the active policy engine, never directly spawn host processes or bypass WorkspaceFS.
- Performance model: Current file operands are eagerly read and concatenated before running; streaming only covers stdin. The runner keeps arrays, pipe/file getline caches, output redirection buffers, and file outputs in memory. `for`/`while` loops have a hard 100,000 iteration guard. Large files, large arrays, long regexes, and command-output getline can cause memory spikes; cancellation and output/file size limits need oracle-backed behavior.
- Oracle/stress gaps: None for command-local coverage after the tests recorded in Closure status; fixture expansion or shared-runtime validation outside this batch is recorded under Deferred with reason.
- Risk: high - awk is a full language, and the current native interpreter is a partial implementation against the mawk lexer/parser/VM surface.
